# Система сверки финансовых транзакций

## Описание проекта

Это тестовое техническое задание. Проект представляет собой автоматизированную систему для сверки финансовых транзакций из двух различных источников: `RpaBank` и `Pindodo`. Система парсит данные из предоставленных отчетов, выполняет их сравнение на основе заданных ключей (дата, сумма, валюта, номер карты, ID терминала) и генерирует подробный Excel-отчет, который включает:

*   Успешно совпавшие транзакции.
*   Транзакции, найденные только в отчете `RpaBank`.
*   Транзакции, найденные только в отчете `Pindodo`.

## Ключевые особенности

*   **Модульность**: Код логически разделен на независимые модули (`parsers.py` для обработки входных данных, `reconcile.py` для выполнения сверки, `export.py` для форматирования и сохранения результатов, `config.py` для централизованной настройки). Это значительно упрощает понимание, тестирование, отладку и поддержку системы.
*   **Масштабируемость**: Использование библиотеки `pandas` для обработки данных обеспечивает высокую производительность и эффективность при работе с большими объемами транзакций. Модульная архитектура позволяет легко добавлять новые источники данных или адаптировать систему под измененные форматы отчетов без существенной переработки существующего кода.
*   **Скорость и Производительность**: Операции по парсингу и сверке оптимизированы за счет использования регулярных выражений и мощных функций `pandas`, что гарантирует быструю обработку данных даже при значительных объемах.
*   **Гибкость конфигурации**: Все ключевые параметры, такие как пути к файлам и поля для сверки, вынесены в отдельный файл `config.py`, что позволяет легко адаптировать систему под различные сценарии использования без изменения основного кода.
*   **Надежное логирование**: Интегрированная система логирования (`loguru`) предоставляет подробную информацию о ходе выполнения скрипта, возможных ошибках и предупреждениях. Это помогает оперативно выявлять и устранять проблемы, а также отслеживать статус обработки.

## Установка

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <https://github.com/hydrauluu/test_tech_task.git>
    cd test_tech_task
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    # Активация (Windows):
    # .\venv\Scripts\activate
    # Активация (macOS/Linux):
    # source venv/bin/activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Подготовьте входные файлы:**
    Убедитесь, что файлы `RpaBank_report.txt` и `Pindodo_report.txt` находятся в соответствующих папках (`RPA/` и `PINDODO/`).

## Использование

Запустите основной скрипт из корневой директории проекта:

```bash
python script/main.py
```

Результаты сверки будут сохранены в файле `Результат/reconciliation.xlsx`. Логи выполнения будут доступны в консоли и в файле `logs/reconciliation.log` (если эта директория и файл будут созданы).
